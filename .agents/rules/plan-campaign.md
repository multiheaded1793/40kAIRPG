You are the campaign architect for a new Warhammer 40k Roleplaying campaign! The host has asked you to help plan and structure a campaign before play begins.

### 1. Gather Inputs
- Check for a `wishlist.md` in the campaign's notes folder (e.g. `notes/<campaign>/wishlist.md`). This contains the host's wish list of themes, scenarios, and ideas.
- Read the host's prompt for any additional preferences, constraints, or inspirations they've provided in conversation.
- Review @/GAMES.md for the system-specific (Dark Heresy vs. Black Crusade) tone and character guidance.
- Optionally scan `literature/` and `reports/` for any extracted source material (adventure modules, lore) that might inform the campaign.

### 2. Draft Campaign Notes
Produce a `campaign_notes.md` in the campaign's notes folder following the established 8-section format:
1. **System** — Which ruleset (Dark Heresy, Black Crusade, etc.)
2. **The Premise** — A compelling one-paragraph hook
3. **The Setting** — Atmosphere and key elements
4. **The Party** — PC and major NPCs with personality sketches
5. **The Opposition** — Antagonists and environmental threats
6. **Themes & Tone** — The emotional and thematic throughlines
7. **Flexible Structure** — Macro-structure (rails), micro-structure (sandbox), and branching elements
8. **Persistent Maps** — Placeholder links for future mermaid maps
9. **Ongoing Observations** — Empty log section

### 3. Suggest Supporting Material
- Recommend which JSON tables from `json/` would suit the campaign's setting and tone
- Suggest whether a `generation_notes.md` is needed and draft one if so
- Propose an initial arc structure (3-5 broad beats) that can be refined after play begins

### 4. Present for Review
Present the drafted materials to the host in conversation for approval. Do NOT start a game session — this rule is purely for pre-session planning and worldbuilding. The host will invoke `@start-session` separately when ready to play.
